# Room Booking Service
Сервис бронирования переговорок для тестового задания Avito.

Реализована обязательная часть задания, а также дополнительные возможности:
- регистрация и логин по email/паролю
- mock conference service
- Swagger
- Makefile
- нагрузочное тестирование
- линтер
- unit / integration tests

## О проекте
Сервис позволяет:

### Admin
- создавать переговорки
- создавать расписание доступности переговорки один раз
- просматривать список всех броней с пагинацией

### User
- просматривать список переговорок
- просматривать доступные слоты по переговорке и дате
- создавать бронь на слот
- отменять свою бронь
- просматривать свои будущие брони

## Основные возможности
- JWT-авторизация
- обязательный `/dummyLogin` для тестовых ролей `admin` и `user`
- отдельные права доступа по ролям
- генерация и хранение слотов в БД
- стабильные `slotId`
- защита от двойного бронирования
- идемпотентная отмена брони
- хранение и передача времени только в UTC
- health endpoint `/_info`
- Swagger UI

## Архитектурные решения

### 1. Слоты материализуются в базе данных
Слоты не генерируются на лету при каждом запросе, а создаются и хранятся в БД.

Это даёт:
- стабильные `slotId`
- быстрый ответ самого нагруженного endpoint `/rooms/{roomId}/slots/list`
- простую проверку занятости слота
- понятную модель для бронирования по `slotId`

### 2. Горизонт генерации слотов
Выбран горизонт ближайших `7` дней.

### 3. Идемпотентная отмена
Повторный вызов отмены уже отменённой брони:
- не считается ошибкой
- возвращает `200 OK`
- возвращает актуальное состояние брони со статусом `cancelled`

### 4. Разделение URL для Docker и локальных тестов
Для корректной работы на Windows и локально:
- приложение в Docker использует подключение к БД через `db:5432`
- локальные repo/integration тесты используют `localhost:5433`

## Запуск проекта
Запуск проекта происходит с помощью команды 

```bash
docker compose up --build
```

Сервис будет доступен по адресу:

- API: `http://localhost:8080`
- Swagger UI: `http://localhost:8080/swagger/index.html`

## Тестирование
Запуск всех тестов:

```bash
go test ./...
```

или 

```bash
make test
```

Запуск тестов с покрытием:

```bash
go test ./... -coverprofile=coverage.out
go tool cover -func coverage.out
```

или 

```bash
make cover 
```

Было достигнуто покрытие тестами выше 70%.

Детали (стек, тесты, структура) вынес в отдельные файлы в папке docs, чтобы не перегружать README.md, улучшить читаемость и упростить ревью

- TESTS.md - тестs и примеры API
- LOAD_TEST.md - нагрузочное тестирование
- DETAILS.md - стек, структура и тд 
- DECISIONS.md - решения, принятые во время разработки
- ARCHITECTURE.md - подробнее об архитектуре
