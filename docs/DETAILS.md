## Основные каталоги
- `cmd/` — точки входа приложения и mock conference service
- `internal/` — основная логика приложения
- `pkg/` — вспомогательные пакеты
- `db/` — SQL-модели и миграции
- `docs/` — swagger и документация
- `scripts/` — вспомогательные скрипты, включая нагрузочный сценарий

## Swagger
Swagger UI доступен после запуска проекта:

```text
http://localhost:8080/swagger/index.html
```

Генерация swagger-документации:

```bash
make swagger
```

## Расписание создаётся один раз и не изменяется
По условиям задания расписание переговорки создаётся только один раз.  
После создания оно считается неизменяемым.

Это упрощает:
- модель данных
- генерацию слотов

## Миграции применяются пофайлово с учётом уже применённых
Используется простая таблица `schema_migrations`, чтобы не выполнять одни и те же миграции повторно.
Миграции реализованы в виде SQL-файлов и применяются собственно реализованным migration runner'ом.
